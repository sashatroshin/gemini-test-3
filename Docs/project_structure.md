# Структура проекта: Онлайн-форма для отчёта по смене

## Корневой каталог

Структура проекта будет максимально простой и плоской, чтобы соответствовать требованиям к статическому сайту.

```
shift-report-form/
├── docs/
│   ├── Implementation.md
│   ├── project_structure.md
│   └── UI_UX_doc.md
├── index.html
├── style.css
├── script.js
├── config.json
└── README.md
```

## Подробная структура

- **`shift-report-form/`**: Корневая папка проекта.

- **`docs/`**: Папка для хранения всей проектной документации.
  - `Implementation.md`: Технический план реализации (этот документ).
  - `project_structure.md`: Описание структуры проекта (текущий документ).
  - `UI_UX_doc.md`: Документация по дизайну и UX.

- **`index.html`**: Основной файл приложения. Содержит всю HTML-разметку для формы, полей ввода и области сводки. 

- **`style.css`**: Файл стилей. Отвечает за весь визуальный вид приложения, включая двухколоночную структуру, стилизацию полей, кнопок и типографику. 

- **`script.js`**: Основной файл с логикой. Здесь будет реализован весь интерактив:
  - Загрузка конфигурации из `config.json`.
  - Обработка ввода и вычисление математических выражений.
  - Динамическое добавление полей расходов.
  - Все расчеты для сводки.
  - Логика бизнес-даты.
  - Сохранение и загрузка данных из `localStorage`.
  - Отправка данных на вебхук.

- **`config.json`**: Файл конфигурации. Хранит все настраиваемые параметры, такие как URL вебхука, категории расходов и шаблоны сообщений. **Не должен** добавляться в систему контроля версий, если содержит чувствительные данные. Рекомендуется использовать `config.example.json`.

- **`README.md`**: Файл с описанием проекта, инструкциями по первой настройке (копирование `config.example.json` в `config.json` и внесение URL вебхука) и развертыванию.
